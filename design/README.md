# Дизайн-референс

Снэпшот дизайна из проекта claude.ai/design «Скриншотер для Kaiten»
(projectId `9fa896d5-174f-42c2-9dc1-04584ce6f90d`), файл
`Screenshotter for Kaiten.dc.html` — экспорт `Скриншотер для Kaiten.dc.html`
на момент 2026-07-15.

Это статичный снимок на момент экспорта, а не живая синхронизация — макет в
claude.ai/design мог измениться. Перед тем как опираться на конкретную деталь
верстки в спорном случае, стоит свериться заново через MCP-инструмент
`DesignSync` (`get_project`/`list_files`/`get_file` с этим `projectId`),
авторизация — через `/design-login`.

## Как читать файл

Разметка — обычный HTML с инлайновыми стилями (тёмная oklch-палитра, Manrope +
JetBrains Mono, зелёный акцент `oklch(0.6 0.15 165)`), открывается напрямую в
любом браузере и визуально читается без дополнительной инфраструктуры.
`<script src="./support.js">` и обёртка `<x-dc>` — служебные для интерактивного
редактора claude.ai/design (подсветка экранов и т.п.), сам `support.js` не
включён в этот снэпшот — он не самодостаточен (ожидает `window.React`,
подгружаемый средой claude.ai/design) и не нужен для чтения вёрстки.

## Экраны в файле

1. Захват области экрана
2. **Аннотирование скриншота** — отдельный экран-редактор (не встроен в
   маленькую карточку выбора действия): тулбар (Курсор/Стрелка/Прямоугольник/
   Эллипс/Текст/Маркер, 5 цветов, Отменить/Вернуть, Отмена/Готово) + канвас.
3. Выбор действия со скриншотом (без редактирования — финальное превью)
4. История скриншотов сессии (пока не реализовано — см. личный `ROADMAP.local.md`)
5. Создание новой задачи
6. Поиск и прикрепление к существующей задаче
7. Настройки
