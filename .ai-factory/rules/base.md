# Project Base Rules

> Проект стартует с нуля, кода ещё нет — ниже зафиксированы конвенции, которые устанавливаются
> для стека Electron + TypeScript + React. Редактируйте по мере роста проекта.

## Naming Conventions

- Файлы: kebab-case (`screenshot-overlay.ts`, `task-form.tsx`).
- React-компоненты: PascalCase внутри файла (`export function TaskForm()`), сам файл — kebab-case.
- Переменные и функции: camelCase.
- Классы и типы/интерфейсы: PascalCase.
- Константы окружения/конфига: UPPER_SNAKE_CASE.

## Module Structure

- `src/main/` — main-процесс Electron: жизненный цикл приложения, трей, глобальные хоткеи, окна,
  доступ к OS API (safeStorage, desktopCapturer).
- `src/renderer/` — React UI: overlay захвата области, форма задачи, окно настроек.
- `src/preload/` — contextBridge-мосты между main и renderer, минимальный и явный API.
- `src/shared/` — типы и константы, общие для main/renderer/preload (например, форма IPC-сообщений,
  типы полей Kaiten).
- `src/kaiten/` — клиент Kaiten API, изолированный от UI и Electron-специфики (чистый TS-модуль,
  тестируемый без запуска Electron).
- `src/platform/` — точка расширения под будущую поддержку macOS: платформенно-специфичные
  реализации (безопасное хранилище, захват экрана) за общим интерфейсом.

## Error Handling

- Ошибки сети/API Kaiten не должны приводить к потере скриншота и введённых данных формы —
  ошибка показывается пользователю с возможностью повторить отправку.
- В main-процессе — явная обработка ошибок IPC-хендлеров, не допускать необработанных
  промисов (`unhandledRejection`) в фоне.
- Пользовательские сообщения об ошибках — на русском, коротко и по делу.

## Logging

- Локальный лог-файл событий приложения (запуск, захват, отправка в Kaiten, ошибки).
- Секреты (API-ключ) никогда не попадают в лог.

## Security

- API-ключ Kaiten хранится только через `safeStorage` (Electron), не в открытом конфиг-файле.
- Настройки без секретов (домен, дефолтные поля, хоткеи) — в обычном JSON-конфиге.
